## 文件定位

這是一份**操作手冊**，定義從接到規格書到功能交付的完整 AI Coding 流程。

```
文件一（LLM 是什麼、會怎樣）← 已在打磨
  ↓ 推導出
本文件（接到規格後，如何用 LLM 開發）← 等文件一定版

文件三（小量情境的 AI Coding）← 預計後續
```

**核心約束**：本文件的每一條規則都必須能追溯到文件一中的某個 LLM 特性或現象。不能有「因為業界都這樣做」的規則——只有「因為 LLM 有某特性，所以必須這樣做」。

## 預計涵蓋的主題

> 以下為初步規劃，待文件一定版後確認。

- 三層混合架構（Context Package / Quality Gate / Execution Checklist）
- Context Package 目錄結構與維護鐵則
- Quality Gate 定義與觸發矩陣
- Execution Checklist（互動前/後）
- Prompt 結構模板（純對話模式 / Agentic 模式）
- Session 管理策略
- 任務拆分原則與粒度選擇
- 執行順序建議
- 測試粒度分工與驗證層次
- 流程規模選擇（已有文件 / 無文件 / 文件失真）
- 追溯回填機制
- 完成標準與檢查清單
