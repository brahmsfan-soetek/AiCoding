# Step 6：前端實作 Prompt 模板

## 用途說明

**何時使用：** 當 Step 5 的前端任務清單（`{程式編號}_frontend_tasks.md`）確認完畢後，在專案 repo 中開啟新的 AI Session 時使用此模板。

**如何使用：**
1. 在專案 repo 中開啟全新 AI Session（不延續前一步驟的對話）。
2. 將下方 Prompt 複製到新的 AI Session。
3. 將所有 `{佔位符}` 替換為實際的程式編號或說明。
4. AI 會先閱讀 `spec/` 目錄下所有文件，再依任務清單一次性完成全部前端實作。

**前置條件：**
- `spec/` 目錄已包含規格統計文件、SA 釐清回覆及前端任務清單等所有文件。
- 前端任務清單（`{程式編號}_frontend_tasks.md`）已完成審閱並確認無誤。
- 後端 API 介面（路由、請求／回應格式）已在任務清單中定義，或已有現成 API 可供串接。

**注意事項：**
- 前後端分開實作，本步驟**僅處理前端**，不修改後端程式碼。
- 規格統計文件為唯一權威來源，如規格統計與其他文件有衝突，以規格統計為準。
- UI 截圖僅供版面排版參考，不得依截圖自行推斷規格統計中未定義的欄位或邏輯。

---

## Prompt 模板

```
請先完整閱讀 spec/ 目錄下的所有文件，再依照前端任務清單實作全部前端程式碼。

【輸入檔案】
請讀取 spec/ 目錄下的所有文件，包含但不限於：
- {程式編號}_規格統計.md（規格統計，唯一權威來源）
- {程式編號}_frontend_tasks.md（前端任務清單，本次實作依據）
- {程式編號}_test_cases.md（測試清單，供實作時參考驗證規則）
{以下視情況增減}
- {其他參考文件}（{用途說明}）

【UI 截圖 / HTML 參考說明】
若 spec/ 目錄中包含 UI 截圖或 HTML 原型，僅供參考版面排版與佈局。
欄位名稱、欄位行為及所有功能邏輯均以規格統計文件為準，
不得依據截圖自行推斷規格統計中未明確定義的欄位或互動邏輯。

【實作指令】
依照 {程式編號}_frontend_tasks.md 中的任務清單，一次性完整實作所有前端項目。
請自行判斷各任務的相依順序後依序完成，不需等待確認。

【約束條件】
1. 規格統計（{程式編號}_規格統計.md）為唯一權威來源。
   - 如規格統計與其他文件有衝突，一律以規格統計為準。
   - 如規格統計與 UI 截圖有衝突，以規格統計為準。
2. 遵循現有專案的程式碼風格、命名慣例與目錄結構，不自行引入新的架構模式。
3. 元件優先使用任務清單中指定的共用元件，不重複造輪。
4. 表單欄位的必填規則、驗證邏輯、預設值及 LOV 選項，均依規格統計中的定義實作。
5. 畫面之間的跳轉、帶參數邏輯及查詢條件，依規格統計中的流程說明實作。
6. 如遇任務清單中有模糊之處，以規格統計的定義為準，並在程式碼中加上說明注解。

【完成後自檢（請逐項確認）】
1. 任務清單中每一個項目是否都已實作完成，無遺漏？
2. 所有欄位的必填、驗證、預設值是否與規格統計一致？
3. LOV / 下拉選單的資料來源與帶入邏輯是否依規格統計定義串接？
4. 共用元件是否依任務清單指定使用，未自行替換？
5. 新增的檔案路徑與命名是否符合現有專案的目錄結構慣例？
```
