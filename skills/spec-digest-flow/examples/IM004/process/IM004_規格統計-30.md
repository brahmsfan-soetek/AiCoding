## 業務概述

庫存異動單支援五種移動類型：

| 代碼 | 名稱 | 說明 | 是否產生傳票 |
|------|------|------|:---:|
| D01 | 庫存調撥單 | 不同工廠/庫位間調撥，總庫存價值不變 | 否 |
| D02 | 成本中心領用 | 庫存物料分配至成本中心，內部消耗追蹤 | 是 |
| D03 | 存貨報廢處理 | 損壞/過期/不可用物料，產生損失記錄 | 是 |
| D04 | 雜收 | 盤盈、退料入庫、免費贈品等庫存增加 | 是 |
| D05 | 雜發 | 盤虧、損耗、樣品贈送等庫存減少 | 是 |

### 不在此程式範圍

- 期末移動平均成本歷史建檔（由 CO003-2 批次作業處理）
- 採購收貨 / 銷貨發料 / 發票驗證等其他 TransactionType 之成本計算

- 開發時 HTML 僅作為 UI 排版參考，實際行為與規格不一致時須確認，不可直接照搬 HTML 實作

## LOV 資料來源

### LOV 2.1 — 物料代碼

| 項目 | 說明 |
|------|------|
| **來源表** | MM_MATERIAL (a), MM_UNIT (mu), MM_MATERIAL_VALUATION (mmv), AU_DATA_PROFILE (adp) |
| **帶入目標** | D.MATERIAL_CODE, D.MATERIAL_DESC(a.MATERIAL_NAME), D.UNIT(a.MAIN_UNIT), D.UNIT_PRICE(mmv.AVG_PRICE), 評價類別顯示用 |
| **AVG_PRICE 為 NULL 時** | 不可選擇該物料 |

**篩選條件 SQL：**
```sql
SELECT a.MATERIAL_CODE, a.MATERIAL_NAME, a.MAIN_UNIT,
       a.ENABLE_BATCH_CONTROL,          -- 啟動批號控制 Y/N
       mu.UNIT_DEC,                     -- 單位小數位數
       mmv.AVG_PRICE,                   -- 單價（顯示小數兩位，NULL 不可選）
       CONCAT(adp.PF_VALUE, ' - ', mmv.VAL_CLASS)  -- 評價類別
  FROM MM_MATERIAL a
  JOIN MM_UNIT mu           ON a.MAIN_UNIT = mu.UNIT_CODE
  JOIN MM_MATERIAL_VALUATION mmv
       ON a.COMPANY_CODE = mmv.COMPANY_CODE
      AND a.MATERIAL_CODE = mmv.MATERIAL_CODE
  JOIN AU_DATA_PROFILE adp
       ON adp.PF_TYPE = 'VAL_CLASS'
      AND mmv.VAL_CLASS = adp.PF_CODE
 WHERE a.STATUS = 'A'
   AND a.COMPANY_CODE = :COMPANY_CODE;   -- 前端傳入
```

**回傳欄位：** MATERIAL_CODE, MATERIAL_NAME, MAIN_UNIT, ENABLE_BATCH_CONTROL, UNIT_DEC, AVG_PRICE, VAL_CLASS 描述

| 欄位 | 說明 |
|------|------|

### 特殊規則 — 即時驗證

| 規則 | 檢查時機 | 行為 |
|------|---------|------|
| **評價類別一致性** | 選擇物料當下 | 同一張異動單中不允許挑選不同 VAL_CLASS 的物料 |
| **庫存水位警告** | 輸入出庫數量當下 | 出庫數量 > 畫面顯示庫存水位時，**前端即時警告**，不送後端（來源：最新規格確認 #3） |
| **後端庫存不足** | 過帳時後端 | 後端檢查不足時交易不完成，**全部回滾**並顯示數量不足訊息（來源：最新規格確認 #4） |
| **併發鎖定** | 開啟暫存單據編輯 | 若單據已被他人編輯中，顯示「此筆單據已在編輯中，不允許同時操作」（來源：最新規格確認 #6） |
| **AVG_PRICE 為 NULL** | LOV 2.1 | 該物料不可被選擇 |

## 過帳邏輯（完整欄位對照）

過帳為單一 LUW，以下步驟依序執行，任一失敗全部回滾。

### 步驟零：前置檢查與狀態更新

1. 以 H.POSTING_DATE + COMPANY_CODE 查詢 GL_PERIOD_STATUSES，確認 GPS.STATUS = 'O'（會計期間開放）
2. 後端再次檢查庫存充足（出庫類型：D01出庫端 / D02 / D03 / D05）
3. 更新 IM_DOC_HEADER：DOC_STATUS → 'P'

### 步驟一：建立 MM_TRANSACTIONS

- D01：每筆明細建立**兩筆**交易（一出一入），其他欄位相同
- D02 / D03 / D04 / D05：每筆明細建立**一筆**交易

### 取得傳票科目邏輯（D1 章節）

每張傳票需取得**兩筆**科目（存貨科目 + 費用科目），借貸方向相反。

**第一筆：存貨科目**

| 步驟 | 動作 | SQL 參考 |
|------|------|---------|
| 1.1 | 取得物料 VAL_CLASS | `SELECT mmv.VAL_CLASS FROM MM_MATERIAL_VALUATION mmv WHERE MATERIAL_CODE = :第一筆物料 AND COMPANY_CODE = :COMPANY_CODE` |
| 1.2.1 | 取得借貸方向與交易碼 | `SELECT amm.DC_INDICATOR, amm.TXN_KEY FROM AU_MVTXN_MAPPING amm WHERE COMPANY_CODE = :COMPANY_CODE AND MVT_TYPE = :MVT_TYPE` |
| 1.2.2 | 取得存貨科目 | `SELECT atk.DR_ACCOUNT, atk.CR_ACCOUNT FROM AU_TXN_KEY atk WHERE COMPANY_CODE = :COMPANY_CODE AND TXN_KEY = :步驟1.2.1之TXN_KEY AND VAL_CLASS = :步驟1.1之VAL_CLASS`；若 DC_INDICATOR = 'C' 取 CR_ACCOUNT，否則取 DR_ACCOUNT |

**第二筆：費用科目**

| 步驟 | 動作 | 說明 |
|------|------|------|
| 2 | 確定 TXN_KEY / SUB_TXN_KEY | D02 或 D05 → TXN_KEY=`'OEI'`, SUB_TXN_KEY=`'IGI'`；D03 → TXN_KEY=`'OEI'`, SUB_TXN_KEY=`'DST'`；D04 → TXN_KEY=`'OEI'`, SUB_TXN_KEY=`'NPR'` |
| 3.1 | DC_INDICATOR 取反 | 若步驟 1.2.1 為 'D' 則此處為 'C'；反之為 'D' |
| 3.2 | 取得費用科目 | `SELECT atk.DR_ACCOUNT, atk.CR_ACCOUNT FROM AU_TXN_KEY atk WHERE COMPANY_CODE = :COMPANY_CODE AND TXN_KEY = :步驟2之TXN_KEY AND SUB_TXN_KEY = :步驟2之SUB_TXN_KEY`；依 3.1 的 DC_INDICATOR 取對應科目 |

> ⚠️ 規格 SQL 範例步驟 3.2 之 TXN_KEY 寫成 `'INP'`（步驟 1.2.1 的值），但註解說「傳入 2 取得之 txn_key」應為 `'OEI'`。此為 SQL 範例與邏輯說明不一致，見不一致 #5。

## 所有操作流程

### 查詢流程

1. 使用者輸入篩選條件 → 按查詢
2. 後端查詢 IM_DOC_HEADER + COUNT/SUM(IM_DOC_DETAIL)
3. 回傳結果列表，依 DOC_STATUS 顯示對應操作按鈕

### 新增 / 編輯流程

1. 使用者選擇異動類型 → 畫面切換對應欄位
2. 填寫表頭：過帳日期、文件日期、條件欄位（成本中心/利潤中心/調撥單號/原因...）
3. 新增明細行：選物料(LOV 2.1) → 帶入名稱/單位/單價/評價類別 → 選工廠(LOV 2.2) → 選庫位(LOV 2.3) → 帶入庫存水位 → （D01）選目標工廠/庫位 → （批次控制）選/輸入批次號 → 輸入數量 → 自動計算金額
4. 前端即時驗證：評價類別一致性、庫存水位警告
5. 按儲存 → 寫入 IM_DOC_HEADER + IM_DOC_DETAIL，DOC_STATUS = 'A'

### 過帳流程

1. 使用者按過帳鈕
2. 前置檢查：會計期間開放、後端庫存充足
3. 更新 IM_DOC_HEADER.DOC_STATUS → 'P'
4. 建立 MM_TRANSACTIONS（D01 兩筆/其他一筆 per 明細）
5. 更新 MM_MATERIAL_STORAGE.AVL_QTY（及 MM_MATERIAL_BATCH.AVL_QTY）
6. 呼叫 usp_UpdateMovingAverageCost（D01 除外）→ 更新 MM_MATERIAL_VALUATION + 寫入 AVGPRICE_LOG
7. 產生傳票 GL_JOURNAL_HEADER + GL_JOURNAL_DETAIL（D01 除外）
8. 更新 AGN 編碼序號
9. 全部成功 → COMMIT；任一失敗 → ROLLBACK

### 刪除流程

1. 查詢頁面，DOC_STATUS = 'A' 時顯示刪除鈕
2. 刪除 IM_DOC_DETAIL + IM_DOC_HEADER

## 命名不一致標記

| # | 位置 | 不一致內容 | 建議 |
|---|------|---------|------|
| 1 | **IM_DOC_DETAIL vs IM_DOC_LINE** | 存取權限表寫 `IM_DOC_LINE`；其餘所有欄位說明、別名表、最新規格確認均使用 `IM_DOC_DETAIL` | **以 `IM_DOC_DETAIL` 為準**（出現頻率高，且最新規格確認也使用 Detail） |
| 2 | **IM_MATERIAL vs MM_MATERIAL** | 最新規格確認 #2 寫「IM_MATERIAL.ENABLE_BATCH_CONTROL」；LOV 2.1 SQL 使用 `MM_MATERIAL` | **以 `MM_MATERIAL` 為準**（LOV SQL 明確使用 MM_MATERIAL，且系統其他 MM 開頭表格一致） |
| 3 | **D02/D03 額外顯示欄位疑似對調** | D02(成本中心領用) 顯示「成本中心、報廢原因」；D03(存貨報廢) 顯示「成本中心、領用申請」——語義不匹配 | **建議對調**：D02 → 成本中心、領用申請(H.REF_DOC_NO)；D03 → 成本中心、報廢原因(H.DESCRIPTION) |
| 4 | **D.POSTING_DATE vs H.POSTING_DATE** | 傳票 PERIOD_YEAR 來源描述寫「以畫面上過帳日期[D.POSTING_DATE]」，但 POSTING_DATE 定義在 IM_DOC_HEADER 而非 DETAIL | **以 `H.POSTING_DATE` 為準** |
| 5 | **科目邏輯步驟 3.2 SQL 的 TXN_KEY 值** | SQL 範例 WHERE 條件寫 `TXN_KEY = 'INP'`，但註解說「傳入 2 取得之 txn_key」，步驟 2 定義 TXN_KEY = `'OEI'` | **SQL 範例應改為 `TXN_KEY = 'OEI'`**（'INP' 是步驟 1.2.1 從 AU_MVTXN_MAPPING 取得的存貨交易碼，步驟 3.2 應使用步驟 2 的費用交易碼 'OEI'） |
| 6 | **JE_LINE_ID 格式範例** | 規格範例寫 `MP100020250000012647001`（以 MP 開頭），但定義說「取 JE_HEADER_ID + 三碼序號」，JE_HEADER_ID 以 JE 開頭 | **範例應改為 `JE10002025000012674001`**（與 JE_HEADER_ID 格式一致） |
| 7 | **AGN 表格名稱** | 規格僅寫 `AGN.PREFIX`、`AGN.CURRENT_NUMBER`，未給出完整表格名稱 | **需確認實際表名**（推測為 `AU_GENERAL_NO`） |
| 8 | **STOCK_TYPE_REDUSE 拼寫** | 欄位名稱 `STOCK_TYPE_REDUSE`（應為 REDUCE） | 若為既有資料庫欄位名則保留，否則建議修正為 `STOCK_TYPE_REDUCE` |
