## 業務概述

庫存異動單支援五種移動類型：

| 代碼 | 名稱 | 說明 | 是否產生傳票 | 成本計算 |
|------|------|------|:---:|:---:|
| D01 | 庫存調撥單 | 不同工廠/庫位間調撥，總庫存價值不變 | 否 | 不呼叫 |
| D02 | 成本中心領用 | 庫存物料分配至成本中心，內部消耗追蹤 | 是 | GI |
| D03 | 存貨報廢處理 | 損壞/過期/不可用物料，產生損失記錄 | 是 | SCRAP |
| D04 | 雜收 | 盤盈、退料入庫、免費贈品等庫存增加 | 是 | MIS_REC |
| D05 | 雜發 | 盤虧、損耗、樣品贈送等庫存減少 | 是 | MIS_DEL |

### 不在此程式範圍

- 期末移動平均成本歷史建檔（由 CO003-2 批次作業處理）
- 採購收貨 / 銷貨發料 / 發票驗證等其他 TransactionType 之成本計算
- ~~庫位 / 批次庫存更新~~（已納入範圍：由 DB 預存程序處理，程式端於過帳流程中呼叫，詳見步驟 3.5）

## 專案實作要求

- 開發時 HTML 僅作為 UI 排版參考，實際行為與規格不一致時須確認，不可直接照搬 HTML 實作
- `COMPANY_CODE`：從登入者 session / ROLE_PROFILE 取得（已有現成實作）
- 後端查詢全都不做分頁和排序；篩選條件全選填，全空查全部
- **資料表前綴規則**：規格中出現 `MM_` / `INV_` 開頭的表名，實際 DB 統一為 `IM_` 開頭

### 前端元件採用

| 類型 | 對應元件 |
|------|---------|
| table | TablesComponent |
| input | SInput2 |

### 後端實作

- 過帳為單一 LUW（Logical Unit of Work），任一步驟失敗全部回滾
- 併發控制：悲觀鎖，兩人可同時開啟同一張暫存單據，但**儲存時行鎖**（SELECT ... FOR UPDATE），後儲存者被阻塞至前者完成
- 刪除方式：**硬刪除**（DELETE），表格無軟刪除欄位
- 審計欄位（CREATOR / CREATE_DT / UPDATER / UPDATE_DT）：由 ORM 自動填入

### 編碼規則

取號表：`AU_GEN_NUMBER`

```sql
CREATE TABLE AU_GEN_NUMBER (
  RANGE_ID      BIGINT AUTO_INCREMENT NOT NULL COMMENT '取號範圍ID',
  FORM_TYPE     VARCHAR(3)  NOT NULL COMMENT '表單名稱',
  PREFIX        VARCHAR(15) NOT NULL COMMENT '前綴字',
  N_YEAR        YEAR        NOT NULL COMMENT '會計年度',
  START_NUMBER  BIGINT      NOT NULL COMMENT '起始編號',
  END_NUMBER    BIGINT      NOT NULL COMMENT '編號上限',
  CURRENT_NUMBER BIGINT     NULL     COMMENT '目前編號',
  CREATOR       VARCHAR(20) NOT NULL,
  CREATE_DT     DATETIME    NOT NULL DEFAULT CURRENT_TIMESTAMP,
  UPDATER       VARCHAR(20) NOT NULL,
  UPDATE_DT     DATETIME    NOT NULL DEFAULT CURRENT_TIMESTAMP,
  COMPANY_CODE  VARCHAR(4)  NOT NULL,
  PRIMARY KEY (RANGE_ID),
  UNIQUE KEY UK1_AU_GEN_NUMBER (FORM_TYPE, PREFIX, N_YEAR)
) ENGINE=InnoDB COMMENT='取號設定檔';
```

取號流程：`SELECT ... FOR UPDATE` → CURRENT_NUMBER + 1 → 檢查 END_NUMBER → UPDATE

| 編碼欄位 | PREFIX 格式 | 序號 | 格式範例 | 備註 |
|---------|-----------|------|---------|------|
| INV_NO | `INV` + YYYYMM | 三碼遞增 | `INV202603001` | 庫存異動單號 |
| TXN_ID | `TXN` + YYYYMM | 三碼遞增 | `TXN202603001` | 庫存交易檔主鍵 |
| JE_NO | `JE` + COMPANY_CODE + PERIOD_YEAR | CURRENT_NUMBER + 1 | `JE10002025000012674` | 傳票號碼 |

## LOV 資料來源

### LOV 2.1 — 物料代碼

| 項目 | 說明 |
|------|------|
| **來源表** | IM_MATERIAL (a), IM_UNIT (mu), IM_MATERIAL_VALUATION (mmv), AU_DATA_PROFILE (adp) |
| **帶入目標** | D.MATERIAL_CODE, D.MATERIAL_DESC(a.MATERIAL_NAME), D.UNIT(a.MAIN_UNIT), D.UNIT_PRICE(mmv.AVG_PRICE), 評價類別顯示用 |
| **AVG_PRICE 為 NULL 時** | 不可選擇該物料 |

**篩選條件 SQL：**
```sql
SELECT a.MATERIAL_CODE, a.MATERIAL_NAME, a.MAIN_UNIT,
       a.ENABLE_BATCH_CONTROL,          -- 啟動批號控制 Y/N
       mu.UNIT_DEC,                     -- 單位小數位數
       mmv.AVG_PRICE,                   -- 單價（顯示小數兩位，NULL 不可選）
       CONCAT(adp.PF_VALUE, ' - ', mmv.VAL_CLASS)  -- 評價類別
  FROM IM_MATERIAL a
  JOIN IM_UNIT mu           ON a.MAIN_UNIT = mu.UNIT_CODE
  JOIN IM_MATERIAL_VALUATION mmv
       ON a.COMPANY_CODE = mmv.COMPANY_CODE
      AND a.MATERIAL_CODE = mmv.MATERIAL_CODE
  JOIN AU_DATA_PROFILE adp
       ON adp.PF_TYPE = 'VAL_CLASS'
      AND mmv.VAL_CLASS = adp.PF_CODE
 WHERE a.STATUS = 'A'
   AND a.COMPANY_CODE = :COMPANY_CODE;   -- 前端傳入
```

**回傳欄位：** MATERIAL_CODE, MATERIAL_NAME, MAIN_UNIT, ENABLE_BATCH_CONTROL, UNIT_DEC, AVG_PRICE, VAL_CLASS 描述

### LOV 2.3 — 庫位（依物料 + 工廠篩選，含庫存水位）

| 項目 | 說明 |
|------|------|
| **來源表** | IM_MATERIAL_STORAGE (mms) |
| **帶入目標** | D.SLOC_CODE, D.STOCK_QTY(mms.AVL_QTY) |

**篩選條件 SQL：**
```sql
-- 出庫類型（D01 出庫端 / D02 / D03 / D05）
SELECT mms.PLANT_CODE, mms.SLOC_CODE, mms.AVL_QTY
  FROM IM_MATERIAL_STORAGE mms
 WHERE mms.COMPANY_CODE = :COMPANY_CODE       -- 前端傳入
   AND mms.MATERIAL_CODE = :MATERIAL_CODE     -- 前端傳入
   AND mms.PLANT_CODE = :PLANT_CODE           -- 前端傳入（連動工廠 LOV）
   AND mms.AVL_QTY > 0;

-- D04（雜收，入庫類型）：移除 AVL_QTY > 0 條件
SELECT mms.PLANT_CODE, mms.SLOC_CODE, mms.AVL_QTY
  FROM IM_MATERIAL_STORAGE mms
 WHERE mms.COMPANY_CODE = :COMPANY_CODE
   AND mms.MATERIAL_CODE = :MATERIAL_CODE
   AND mms.PLANT_CODE = :PLANT_CODE;
```

**回傳欄位：** PLANT_CODE, SLOC_CODE, AVL_QTY

| 步驟 | 動作 | SQL 參考 |
|------|------|---------|
| 1.1 | 取得物料 VAL_CLASS | `SELECT mmv.VAL_CLASS FROM IM_MATERIAL_VALUATION mmv WHERE MATERIAL_CODE = :第一筆物料 AND COMPANY_CODE = :COMPANY_CODE` |

| 步驟 | 動作 | 說明 |

## 所有操作流程

### 查詢流程

1. 使用者輸入篩選條件（全選填）→ 按查詢
2. 後端查詢 IM_DOC_HEADER + COUNT/SUM(IM_DOC_DETAIL)
3. 回傳結果列表，依 DOC_STATUS 顯示對應操作按鈕

### 新增 / 編輯流程

1. 使用者選擇異動類型 → 畫面切換對應欄位
2. 填寫表頭：過帳日期、文件日期、條件欄位（D01:調撥單號 / D02:成本中心+領用申請 / D03:成本中心+報廢原因 / D04:利潤中心+收貨原因 / D05:利潤中心+發貨原因）
3. 新增明細行：選物料(LOV 2.1) → 帶入名稱/單位/單價/評價類別 → 選工廠(LOV 2.2) → 選庫位(LOV 2.3) → 帶入庫存水位 → （D01）選目標工廠/庫位 → （批次控制）選/輸入批次號 → 輸入數量 → 自動計算金額
4. 前端即時驗證：評價類別一致性、庫存水位警告
5. 按儲存 → 悲觀鎖（行鎖）→ 寫入 IM_DOC_HEADER + IM_DOC_DETAIL，DOC_STATUS = 'A'

### 過帳流程

1. 使用者按過帳鈕
2. 前置檢查：會計期間開放、後端庫存充足（檢查 IM_MATERIAL_VALUATION.STOCK_QTY）
3. 更新 IM_DOC_HEADER.DOC_STATUS → 'P'
4. 建立 IM_TRANSACTIONS（D01 兩筆/其他一筆 per 明細）
5. 更新 IM_MATERIAL_VALUATION + 寫入 IM_MATERIAL_AVGPRICE_LOG（D01 除外）
6. 產生傳票 GL_JOURNAL_HEADER + GL_JOURNAL_DETAIL（D01 除外）
7. 呼叫 DB 預存程序更新庫位/批次庫存（IM_MATERIAL_STORAGE / IM_MATERIAL_BATCH）
8. 更新 AU_GEN_NUMBER 編碼序號
9. 全部成功 → COMMIT；任一失敗 → ROLLBACK

### 刪除流程

1. 查詢頁面，DOC_STATUS = 'A' 時顯示刪除鈕
2. 硬刪除 IM_DOC_DETAIL + IM_DOC_HEADER
