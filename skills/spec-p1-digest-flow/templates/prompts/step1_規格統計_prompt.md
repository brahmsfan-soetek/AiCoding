# Step 1：規格統計 Prompt 模板

## 用途說明

**何時使用：** 當你從 SA 收到規格書後，執行此步驟。

**如何使用：**
1. 將所有 `{佔位符}` 替換為實際檔名或說明。輸入檔案直接從工作目錄讀取。
2. AI 產出的規格統計文件即為後續所有開發步驟的**唯一權威來源（Single Source of Truth）**，請存檔保存。

---

## Prompt 模板

```
根據以下參考資料，依照規格統計模板的結構產出一份完整的規格統計文件。
本文件將作為後續所有開發步驟的唯一權威來源（Single Source of Truth），
請務必完整、準確，不得遺漏任何規格細節。

【輸入檔案】
- {主規格檔名}（主規格）
- {規格修正檔名}（規格修正，優先級高於主規格，如有衝突以此為準）
{以下視情況增減}
- {參考規格 1}（{用途說明}）
- {參考規格 2}（{用途說明}）

【UI 截圖 / HTML 參考說明】
若提供了 UI 截圖或 HTML 原型，僅供參考版面排版與佈局，
欄位名稱、欄位行為及所有功能邏輯均以規格書文字內容為準，
不得依據截圖自行推斷未在規格書中明確定義的欄位或邏輯。

【輸出要求】
1. 依照規格統計模板的結構產出（不足處可新增或調整章節，但不得刪減已定義章節）。
2. 涉及查詢、寫入的業務邏輯，須列出每張目標表的完整欄位對照，包含：
   - 欄位名（資料庫欄位名稱）
   - 必填（是 / 否 / 條件必填，請說明條件）
   - 輸入值（畫面上用戶輸入或系統帶入的值）
   - 寫入值（實際寫入資料庫的值）
   - 來源說明（值的來源，例如：畫面輸入、系統日期、LOV 帶入、固定值等）
3. 每個 LOV / 下拉選單須獨立列出：
   - 來源表
   - 篩選條件（含對應 SQL 或等效邏輯）
   - 回傳欄位
   - 帶入目標欄位
4. 表名或欄位名在規格中前後不一致時，標記不一致之處並建議以哪個為準。
5. 列出規格中實際使用但未出現在存取權限表中的所有表格。
6. 編碼規則若規格未定義，標記「⚠️ 未定義」。

【完成後自檢（請逐項確認）】
1. 是否有任何欄位在原始規格中存在，但統計中遺漏了？
2. 每個 CRUD 操作涉及的表格，是否都有完整欄位對照？
3. 規格修正檔的所有修正是否已完整反映到統計中？
```
