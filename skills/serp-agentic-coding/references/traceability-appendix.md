# 01 特性速查與 Phase 對應

## 附錄 A：11 項 LLM 特性 → Phase 防禦

| # | 特性 | 核心數據 | 防禦所在 Phase |
|---|------|---------|---------------|
| 1 | 語義漂移 | 串接產出收斂到模型偏好，非原始意圖 | P1（從源頭載入）、P2（每 Task 重載） |
| 2 | Context Rot | 策劃 context 減少 60% 錯誤；~3K tokens 推理退化 | P2（精簡 context 載入規則） |
| 3 | 位置偏見 | MRCR v2: Opus 76%, Sonnet 18.5% | 所有 Phase 一律用最強模型 |
| 4 | 語義競爭 | 矛盾 context → 100% 靜默混合 | **P0（衝突檢查前置）** |
| 5 | 模式複製 | Bug-identical 44.44%; test oracle <50% | **P1（從規格寫測試）**、P2（正確範例 + 模板） |
| 6 | 自信填補 | 19.7% 套件不存在；grounding 提升至 87% | P0（SA 待確認）、P2（驗證存在性） |
| 7 | 附和偏見 | 95% 偏好附和（vs 基線真實回覆） | **P1（反附和檢查）**、**P3（主動找問題 + Wait）** |
| 8 | 自我修正盲區 | 自我 18% vs 外部 88%; Wait 降低盲區率 89.3% | **Agent 分離模型** + P2/P3 Wait 提示 |
| 9 | 規格博弈 | RE-Bench 30.4% vs HCAST 0.7% | P2（測試唯讀 + 退出選項） |
| 10 | 指令飽和 | P(全遵守) = P(單條)^N | SKILL 按 Phase 分段載入 |
| 11 | 複雜度天花板 | >50 行正確率 26.5%；分解 +35-76% | **P1（任務拆解 <50 行）** |

## 附錄 B：交叉影響防禦

| 風險組合 | 放大效果 | 防禦機制 |
|---------|---------|---------|
| §5 + §8（模式複製 + 盲區） | AI 複製 bug 到測試，且無法自我發現 | P1/P2 Agent 分離 |
| §4 + §6（語義競爭 + 幻覺） | 矛盾混合後缺口用幻覺填補 | P0 衝突前置 |
| §1 + §2（漂移 + Context Rot） | 多輪偏差 + context 膨脹 = 雙重退化 | 每 Task 獨立 context |
| §7 + §8（附和 + 盲區） | 審查時附和自己的代碼 | P3 獨立 Review Agent |
| §9 + §5（博弈 + 複製） | 學到「通過測試」的捷徑並複製 | 測試唯讀 + 最小權限 |
| §10 + §11（飽和 + 天花板） | 指令太多 + 任務太大 | Phase 分段 + 任務拆解 |
